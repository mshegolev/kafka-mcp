# Milestone v1.0 — kafka-mcp — Audit Report

## Definition of Done

A read-only Kafka MCP brick that lets investigators find events by business key
(msisdn / customerId / productId / orderId) within a time window and retrieve
decoded message bodies (Avro / Protobuf / JSON via Schema Registry), exposed
identically across four inbound faces (lib `KafkaClient`, MCP stdio, FastAPI
REST, CLI), with a structurally-enforced read-only guarantee, a benchmark-gated
decision on a native accelerator, multi-platform CI, and Glama/PyPI distribution
artifacts.

## Cross-Phase Integration

Integration checker result: **0 orphaned, 0 missing** across 18+ cross-phase
connections. All 16 face×operation pairs (4 faces × {list_topics, describe_topic,
search_messages, get_message}) are WIRED and delegate to the same
`KafkaClient` / `TopicService`. The decode pipeline (Phase 2) is invoked by the
search/get flow over the Phase 1 consumer adapter; Evidence fields flow through
to serialized output on every face; the Phase 3 scanner seam preserves the
read-only guarantee (no produce/commit/subscribe).

### End-to-End Flows (4/4 complete)

1. `list_topics` — all four faces → KafkaClient → TopicService → ConsumerPort → adapter.
2. `describe_topic` — partition ids + watermark offsets → TopicInfo, TopicNotFoundError on miss.
3. `search_messages` — offsets_for_times seek → bounded assign-based scan → resilient decode → Evidence extraction → key/header/value matching → serialized (base64 raw).
4. `get_message` — exact-offset fetch → strict decode (typed DecodeError) → MessageNotFoundError / TransientError mapped per face.

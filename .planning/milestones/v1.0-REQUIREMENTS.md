# Requirements: kafka-mcp

## Overview

Read-only Kafka MCP brick. Enables search of events by key
(msisdn/customerId/productId/orderId) in a time window, retrieval of message
bodies with Avro/Protobuf/JSON decode, and topic inspection. Library-first:
`from kafka_mcp import KafkaClient` works in pytest without MCP/FastAPI.

Part of Investigation MCP Suite (umbrella: `/opt/develop/aiqa/investigate-suite/`).
Full design spec: `/opt/develop/aiqa/docs/superpowers/specs/2026-06-05-investigation-mcp-suite-design.md` (§4.1).

## v1 Requirements

### KAFKA-01: list_topics
Tool `list_topics` returns the list of topic names available on the broker.

### KAFKA-02: search_messages
Tool `search_messages` finds messages by key (msisdn / customerId / productId /
orderId) across specified topics within a time window, returning up to `limit`
`KafkaMessage` objects.

API contract (Investigator Contract):
```python
list[KafkaMessage] = c.search_messages(
    key: str, *, key_field: str | None = None,
    topics: list[str] | None = None,
    time_from: datetime | None = None,
    time_to: datetime | None = None,
    limit: int = 500)
```

### KAFKA-03: get_message
Tool `get_message` retrieves a single `KafkaMessage` by topic / partition /
offset.

### KAFKA-04: describe_topic
Tool `describe_topic` returns partition count, per-partition earliest/latest
offsets, and leader metadata for a named topic.

### KAFKA-05: Avro / Protobuf / JSON decode via Schema Registry
All message values (and optionally keys) are decoded through Schema Registry
before being returned. Supported wire formats: Avro, Protobuf, plain JSON.
Decoded result is exposed as `value: dict` in `KafkaMessage`.

### KAFKA-06: Read-only guarantee
All broker access uses a temporary consumer-group (never commits offsets to
production groups). No produce operations. Scan is bounded (offset range cap,
configurable `limit`). All MCP tool annotations carry `readOnlyHint: true`.

### KAFKA-07: Rust partition scanner (benchmark-gated)
A `native/` Rust/pyo3 scanner for the CPU-bound partition-scan hot-path.
Enabled ONLY after a pytest-benchmark run proves CPU-bound speedup over
pure-Python; pure-Python scan is the default and permanent fallback.
Build via maturin; CI wheels via cibuildwheel.

## Out of Scope (v1)

- **Produce**: writing messages to any topic.
- **Consumer-group management**: listing, resetting, or deleting consumer groups.
- **Exactly-once semantics**: transactional producers/consumers.
- **Broker configuration**: topic creation, ACL management, broker-level ops.
- **Schema Registry write**: registering or evolving schemas.

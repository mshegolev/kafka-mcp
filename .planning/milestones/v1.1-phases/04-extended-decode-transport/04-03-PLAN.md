<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Network → /mcp endpoint | New HTTP surface; any client on the network can send MCP requests |
| server.json/glama.json declaration | Client reads declared URL and connects to it; URL must be accurate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | Information Disclosure | HTTP /mcp endpoint — no auth | accept (v1.1) | The HTTP transport ships WITHOUT authentication by design for v1.1. The run posture is localhost/trusted-network (server.py default bind is 127.0.0.1). HTTP auth/TLS hardening is explicitly deferred to HTTP-02 (Future milestone). The server.py startup warning (already present) fires when KAFKA_MCP_HOST is set to a non-loopback address. This risk is accepted for v1.1 under the trusted-network run posture; it is NOT silently omitted. |
| T-04-08 | Elevation of Privilege | /mcp endpoint — read-only guarantee | mitigate | The FastMCP instance mounted at /mcp is created via create_mcp_server(), which registers all four tools with ToolAnnotations(readOnlyHint=True). The structural read-only guarantee (assign-only consumer, enable.auto.commit=false) is enforced at the KafkaClient / ConfluentConsumerAdapter layer regardless of transport. The HTTP transport cannot introduce write paths. |
| T-04-09 | Spoofing | server.json declares a URL pointing to /mcp | accept | The declared URL (http://localhost:8000/mcp) matches the actual mount path. A misconfigured server.json pointing to a non-existent route would cause client connection failures, not a security issue. The acceptance criterion verifies the URL ends with /mcp. |
| T-04-10 | Denial of Service | /mcp endpoint — MCP request flood | accept | Rate limiting and request size limits are not added in this plan (same posture as existing /tools/* routes). The localhost default bind limits exposure. DoS hardening is deferred with HTTP-02. |
| T-04-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan. FastMCP is already in dependencies (mcp>=1.27). |
</threat_model>

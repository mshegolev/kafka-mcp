  Fixtures to implement:

  1. **`kafka_container`** (session scope):
     - `from testcontainers.kafka import KafkaContainer`
     - Use `KafkaContainer("confluentinc/cp-kafka:7.6.0")` (pinned image tag for reproducibility).
     - Wrap in try/except to catch Docker connection errors; if Docker unavailable,
       call `pytest.skip("Docker not available — skipping integration tests")`.
     - `yield container` after `.start()`, `.stop()` in finally.

  2. **`sr_container`** (session scope, depends on `kafka_container`):
     - Use `DockerContainer("confluentinc/cp-schema-registry:7.6.0")` from `testcontainers.core.container`.
     - Set env vars via `.with_env(...)`:
       - `SCHEMA_REGISTRY_HOST_NAME=schema-registry`
       - `SCHEMA_REGISTRY_KAFKASTORE_BOOTSTRAP_SERVERS=PLAINTEXT://{kafka_internal_host}:{kafka_internal_port}`
     - The kafka internal bootstrap address: use `kafka_container.get_bootstrap_server()` but
       note that SR connects from within the Docker network. The KafkaContainer in testcontainers-python
       exposes `get_bootstrap_server()` for external use. For the SR container, use the
       container's internal network address. Get the kafka container's network info:
       ```python
       kafka_host = kafka_container.get_docker_client().bridge_ip(kafka_container._container.id)
       ```
       OR simpler: use the `with_network()` approach or just use the Docker host networking.
       
       **Recommended approach:** Use `kafka_container.get_bootstrap_server()` which returns
       `host:port` for external access. Since both containers share the Docker host bridge
       network by default in testcontainers, the SR container can reach Kafka at the mapped port
       via the Docker host IP. Use:
       ```python
       # Internal broker address for SR
       kafka_bootstrap = kafka_container.get_bootstrap_server()
       # SR needs the PLAINTEXT:// prefix
       sr_kafka_url = f"PLAINTEXT://{kafka_bootstrap}"
       ```
       Actually, testcontainers maps ports to localhost. For container-to-container communication,
       use the container's internal IP. The simplest approach that works reliably:
       
       Use `with_network()` to put both on the same Docker network, OR use host networking.
       
       **Simplest reliable approach:** After kafka_container starts, get its container internal IP
       and the INTERNAL port (9092 for CP-Kafka):
       ```python
       kafka_network_ip = kafka_container.get_docker_client().bridge_ip(
           kafka_container._container.id
       )
       # CP-Kafka internal listener is on 9092 (PLAINTEXT) by default
       sr_kafka_bootstrap = f"PLAINTEXT://{kafka_network_ip}:9092"
       ```
       
       If `bridge_ip` is unavailable, fall back to inspecting the container:
       ```python
       import docker
       docker_client = docker.from_env()
       container_info = docker_client.containers.get(kafka_container._container.id)
       kafka_network_ip = container_info.attrs['NetworkSettings']['IPAddress']
       # fallback: check bridge network
       if not kafka_network_ip:
           networks = container_info.attrs['NetworkSettings']['Networks']
           kafka_network_ip = list(networks.values())[0]['IPAddress']
       ```
     - Expose port 8081 via `.with_exposed_ports(8081)`.
     - `.start()`, then wait for SR to become healthy:
       ```python
       import urllib.request, time
       sr_url = f"http://{sr_container.get_container_host_ip()}:{sr_container.get_exposed_port(8081)}"
       for _ in range(30):
           try:
               urllib.request.urlopen(f"{sr_url}/subjects", timeout=2)
               break
           except Exception:
               time.sleep(1)
       else:
           pytest.fail("Schema Registry did not become healthy in 30s")
       ```
     - `yield sr_container` after healthy, `.stop()` in finally.

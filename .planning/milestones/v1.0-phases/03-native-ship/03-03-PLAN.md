         github.event_name == 'workflow_dispatch'

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI workflow → PyPI | Token passes only via GitHub Secrets; never stored in repo |
| cibuildwheel matrix → native build | Conditional on native/Cargo.toml presence; cannot silently engage Rust toolchain |
| publish trigger | Release event or workflow_dispatch only; never on every push |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03-A | Tampering | ci.yml action versions | mitigate | Pin all action references to @v4 / @v5 / @v2 major version tags (not @main, not SHA-less); document that SHA pinning is the stronger option but major-version tags are acceptable for this project tier |
| T-03-03-B | Information Disclosure | PYPI_TOKEN | mitigate | Token only in ${{ secrets.PYPI_TOKEN }}; never in plaintext; publish job uses environment: pypi with manual-approval gate recommendation |
| T-03-03-C | Elevation of Privilege | publish trigger | mitigate | publish job condition: github.event_name == 'release' OR 'workflow_dispatch'; protected by GitHub environment approval; not triggered on push to branch |
| T-03-03-D | Tampering | cibuildwheel Rust toolchain | mitigate | cibuildwheel build-wheels job gated on if: hashFiles('native/Cargo.toml') != ''; Rust toolchain is never installed silently for the pure-Python outcome |
| T-03-03-SC | Tampering | npm/pip/cargo installs | accept | No new runtime packages installed; yaml package used only for validation in verify step (stdlib fallback available) |
</threat_model>

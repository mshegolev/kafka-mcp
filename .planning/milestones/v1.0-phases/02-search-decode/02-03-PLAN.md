    GREEN implementation in ConfluentConsumerAdapter.fetch_messages():
    Signature: fetch_messages(self, topic: str, partition: int, start_offset: int,
        stop_offset: int, time_to: datetime | None, limit: int) -> list[KafkaMessage]

    Algorithm:
    1. Build TopicPartition(topic, partition, start_offset) and call self._consumer.assign([tp])
    2. Set a per-call scan counter to 0
    3. Loop: msg = self._consumer.poll(timeout=self._settings.poll_timeout)
       - If msg is None: break (no more messages)
       - If msg.error(): break (transient error — do not raise; caller handles empty result)
       - If msg.offset() >= stop_offset: break
       - scan counter += 1; if scan counter > self._settings.max_scan: break (KAFKA_MCP_MAX_SCAN)
       - Extract timestamp: msg.timestamp() returns (TIMESTAMP_TYPE, milliseconds_int)
         TIMESTAMP_CREATE_TIME = 0 or confluent_kafka.TIMESTAMP_CREATE_TIME
         If timestamp_type == TIMESTAMP_CREATE_TIME and ms > 0:
           ts_utc = datetime.fromtimestamp(ms / 1000.0, tz=timezone.utc)
         Else: ts_utc = datetime.now(tz=timezone.utc)  # fallback — clock-skew note
       - If time_to is not None and ts_utc > time_to: break
       - Extract key: msg.key() bytes → decode as UTF-8 best-effort (errors="replace"); None if key() is None
       - Extract headers: msg.headers() returns list[(name: str, value: bytes)] or None;
         convert to dict[str, str] via utf-8 decode of values (errors="replace"); {} if None
       - Build KafkaMessage(topic=topic, partition=partition, offset=msg.offset(),
           key=key_str, headers=headers_dict, value=None, timestamp_utc=ts_utc, raw=msg.value() or b"")
       - Append to result list
       - If len(result) >= limit: break  (global limit check — also enforced in service)
    4. Return result list

    Do NOT call self._consumer.commit() anywhere. assign() already used (no subscribe).
    Import from datetime: datetime, timezone (stdlib only).
    Import KafkaMessage from kafka_mcp.domain.models.
    Import confluent_kafka.TIMESTAMP_CREATE_TIME constant.
  </behavior>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kafka broker → ConfluentConsumerAdapter | Untrusted wire bytes arrive here; consumer must not commit, not subscribe, and must be bounded |
| fetch_messages scan loop | Unbounded scan is a DoS vector; per-partition max_scan cap + global limit both enforced |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03-A | Denial of Service | fetch_messages scan loop | mitigate | Loop terminates on: stop_offset, limit, max_scan (KAFKA_MCP_MAX_SCAN per partition), time_to exceeded, msg=None (end of partition), or error. All five exit conditions tested. |
| T-02-03-B | Denial of Service | Consumer.poll timeout | mitigate | poll_timeout from KafkaMcpSettings (default 1.0s); fetch_message uses 5× timeout for single-message; no infinite poll loops |
| T-02-03-C | Tampering | Read-only guarantee | mitigate | subscribe() absent from source file (automated test_no_subscribe_in_source verifies); enable.auto.commit=False already set in constructor (Phase 1); fetch_messages and fetch_message use assign() not subscribe() |
| T-02-03-D | Information Disclosure | key/header bytes UTF-8 decode | mitigate | Key and header bytes are decoded with errors="replace" — malformed UTF-8 sequences produce replacement characters (U+FFFD), never raise exceptions or expose raw bytes in error messages |
| T-02-03-E | Denial of Service | Message raw bytes unbounded size | accept | Message size is bounded by broker configuration (message.max.bytes); we consume what the broker provides; scan limit bounds the count; single oversized message returned as-is for caller's decision |
| T-02-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages; confluent-kafka already installed (Phase 1) |
</threat_model>

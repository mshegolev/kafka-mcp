<objective>
Lock the existing mTLS config wiring behind an asserting test on the AdminClient
path (MTLS-01) and document mTLS setup for operators in the README (MTLS-03).

The implementation ALREADY EXISTS: `ConfluentConsumerAdapter.__init__`
(src/kafka_mcp/adapters/outbound/confluent_consumer.py) maps `settings.ssl_*`
into librdkafka `ssl.*` keys on BOTH the Consumer conf and the AdminClient conf,
guarded per-field, with the key password unwrapped inline. `KafkaMcpSettings`
(src/kafka_mcp/config.py) already exposes `ssl_certificate_location`,
`ssl_key_location`, `ssl_ca_location`, `ssl_key_password` (SecretStr), and
`security_protocol`.

The gap this plan closes: existing unit tests in tests/test_adapters.py
(`test_ssl_cert_keys_added_when_fields_set`, `test_ssl_cert_keys_omitted_when_fields_unset`)
capture ONLY the Consumer conf — they patch `AdminClient` with a bare
`return_value=MagicMock()`, so the ssl.* wiring on the AdminClient conf is
unasserted. Per D (Claude's discretion), the unit test must assert ssl.* keys on
BOTH consumer and admin conf, and their absence when unset. This plan adds the
AdminClient-conf assertions and the README mTLS section. It does NOT re-implement
any config or adapter code.

Purpose: prove MTLS-01 (config->librdkafka ssl.* wiring on consumer AND admin,
emitted only when set, no PLAINTEXT/SASL regression) and deliver MTLS-03 (README
mTLS docs).
Output: extended AdminClient-conf assertions in tests/test_adapters.py; an mTLS
subsection in README.md under Configure.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator env -> librdkafka conf | SSL cert/key/CA paths and key passphrase cross from environment into the client conf dict |
| brick -> broker (mTLS) | client-cert TLS handshake replaces PLAINTEXT/SASL trust |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information Disclosure | ssl.key.password in conf/logs | mitigate | Already SecretStr in config.py; conf dict is local to __init__, never stored/logged. Test asserts value is unwrapped only into the local conf. README warns never to commit key/passphrase. |
| T-11-02 | Spoofing | broker identity | accept | ssl.ca.location pins the CA that verifies the broker; verification is librdkafka's responsibility (not re-implemented here). Documented in README. |
| T-11-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installs in this plan (test + docs only); no legitimacy checkpoint required. |
</threat_model>

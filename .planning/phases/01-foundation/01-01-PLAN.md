## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env → KafkaMcpSettings | SASL credentials and SR password enter via environment variables |
| config.py → confluent_kafka | SASL username/password passed as librdkafka config dict |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | KafkaMcpSettings.sasl_password, sr_pass | mitigate | Use pydantic SecretStr for sasl_password and sr_pass; __repr__/__str__ will not expose value; never log the Settings object directly |
| T-01-02 | Tampering | domain/models.py boundary | mitigate | CI ruff lint + hexagonal boundary assertion in test_domain.py catches I/O imports in domain/ at test time |
| T-01-03 | Denial of Service | KafkaMcpSettings.max_scan | mitigate | max_scan field validated > 0 and <= 10_000_000 via pydantic validator in plan 01-02 when consumer is wired; default 100_000 prevents unbounded scans from day 0 |
| T-01-SC | Tampering | pip install (pyproject.toml deps) | mitigate | All packages (mcp, confluent-kafka, fastapi, pydantic, orjson) are well-known, high-download packages; no [ASSUMED]/[SUS] entries — slopcheck gate satisfied |
</threat_model>

    Imports: KafkaMcpSettings from kafka_mcp.config;

    class KafkaClient:

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → KafkaClient.from_env() | Environment variables enter as broker/SR credentials |
| TopicService → ConsumerPort | Domain service calls outbound port — boundary enforced by protocol typing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | KafkaClient.from_env() — logs ConfigError | mitigate | ConfigError message must name only the missing key name, never the value; callers must not log the exception's args chain as it could contain partial config |
| T-03-02 | Tampering | domain/search_service.py hexagonal boundary | mitigate | Automated assertion in tests/test_lib.py: grep for I/O imports in domain/ directory; CI will catch boundary violations on every commit |
| T-03-03 | Elevation of Privilege | TopicService.describe_topic — partition enumeration | mitigate | get_partition_ids scoped to the named topic only; no cross-topic metadata exposure; TopicNotFoundError prevents describing non-existent topics |
| T-03-04 | Denial of Service | TopicService.describe_topic — describe topic with many partitions | mitigate | get_partition_ids returns actual partition list from broker metadata; partition count is bounded by broker config (typically 1–1024); no unbounded loop |
| T-03-SC | Tampering | hatchling + test toolchain | mitigate | All dev tools (pytest, ruff) are well-known; no [ASSUMED]/[SUS] packages |
</threat_model>

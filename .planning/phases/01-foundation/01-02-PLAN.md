<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| KafkaMcpSettings → librdkafka config dict | SASL credentials cross into confluent_kafka.Consumer() |
| ConfluentConsumerAdapter → Kafka broker | Network call; broker may return large topic metadata |
| SchemaRegistryHttpAdapter → Schema Registry HTTP | Credentials passed as Basic-Auth header |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | ConfluentConsumerAdapter — sasl_password in librdkafka conf dict | mitigate | Extract via SecretStr.get_secret_value() immediately before passing to Consumer(); the conf dict is local and never stored or logged; no repr of conf dict in logs |
| T-02-02 | Information Disclosure | SchemaRegistryHttpAdapter — sr_pass | mitigate | Phase 1 stub never makes HTTP calls; password stored in attribute but never logged; Phase 2 must use httpx BasicAuth and not log the Authorization header |
| T-02-03 | Tampering | subscribe() absent from ConfluentConsumerAdapter | mitigate | Automated check in test_adapters.py and CI lint: grep for subscribe() in adapter; assign()-only approach is the structural read-only guarantee (KAFKA-06, D-05) |
| T-02-04 | Denial of Service | list_topics — broker returns tens of thousands of topics | mitigate | list_topics result is a filtered/sorted list; no pagination needed at this layer; max_scan cap applies at scan time in plan 01-03 |
| T-02-05 | Elevation of Privilege | group.id uuid uniqueness | mitigate | uuid4() called per ConfluentConsumerAdapter instantiation; throwaway groups cannot collide with production groups by name pattern kafka-mcp-ro-* |
| T-02-SC | Tampering | pip install confluent-kafka | mitigate | confluent-kafka is a well-established PyPI package (Confluent, Inc.); no [ASSUMED]/[SUS] flag |
</threat_model>

    GREEN phase — implement SchemaRegistryHttpAdapter:

    Constructor signature: __init__(self, url: str | None, user: str | None = None, password: str | None = None)
    If url is not None: instantiate confluent_kafka.schema_registry.SchemaRegistryClient with conf dict:
      {"url": url} plus basic.auth.user.info = f"{user}:{password}" when both provided.
    Store client as self._client (SchemaRegistryClient instance or None).
    Store self._avro_deserializer = None (lazy init) and self._proto_deserializer = None (lazy init).
    Do NOT store password as a plain attribute — pass it directly to the conf dict only.

    decode(raw: bytes, topic: str = "", partition: int = 0, offset: int = 0) -> dict | None
    Note: topic/partition/offset are for DecodeError context only (passed from message metadata).

    Framing detection logic:
    1. If len(raw) < 5 or raw[0] != 0x00 → no Confluent framing → attempt json.loads(raw)
       - If json.loads succeeds → return the dict
       - If json.loads raises → raise DecodeError(topic, partition, offset, reason="json decode failed: {exc}")
    2. If raw[0] == 0x00 → Confluent framing:
       - schema_id = int.from_bytes(raw[1:5], "big")
       - remainder = raw[5:]
       - Look up schema via self._client.get_latest_version(subject) — but we have schema_id, so use
         self._client.get_schema(schema_id) (SchemaRegistryClient.get_schema(schema_id: int) returns RegisteredSchema)
       - registered_schema.schema.schema_type is "AVRO" or "PROTOBUF"
       - For AVRO: use confluent_kafka.schema_registry.avro.AvroDeserializer initialized with
         self._client and registered_schema.schema. Call deserializer(remainder, ctx=None) → dict.
         Wrap any exception in DecodeError.
       - For PROTOBUF: use confluent_kafka.schema_registry.protobuf.ProtobufDeserializer.
         Call deserializer(remainder, ctx=None) → dict (via MessageToDict). Wrap exceptions in DecodeError.
       - For any other schema_type: raise DecodeError(topic, partition, offset, reason=f"unknown schema type: {schema_type}")
    3. If self._client is None (url=None): raise DecodeError(topic, partition, offset, reason="Schema Registry not configured")

    get_schema() remains as-is (returns None — not used in Phase 2 decode pipeline,
    kept for SchemaRegistryPort backward compatibility).

    Security: sr_pass from config is passed directly as conf["basic.auth.user.info"] value and
    NOT stored as instance attribute. __repr__ must not expose it (default pydantic/dataclass repr
    not used — plain class so repr is the default object repr which shows no attributes).
  </behavior>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Kafka broker wire → SchemaRegistryHttpAdapter | Untrusted decoded payloads arrive here; malicious/corrupt Avro or Protobuf must not crash the process or enable code execution |
| SchemaRegistryHttpAdapter → Schema Registry | Credentials (sr_pass) cross this boundary; basic-auth must not appear in logs |
| adapter → domain service | decode() returns dict | None; adapter must never return an object that causes remote code execution in the service layer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-02-A | Tampering | decode() — untrusted payload | mitigate | All deserialization wrapped in try/except; any exception from AvroDeserializer/ProtobufDeserializer/json.loads is caught and re-raised as typed DecodeError — never propagated as unhandled; no eval() or exec() anywhere in decode path |
| T-02-02-B | Information Disclosure | sr_pass in SR conf dict | mitigate | sr_pass extracted via SecretStr.get_secret_value() and passed directly to SchemaRegistryClient conf dict; NOT stored as instance attribute; plain class repr shows no attribute values (T-01-01 established pattern) |
| T-02-02-C | Denial of Service | decode() on oversized payloads | mitigate | Decode is bounded per-message; message scan limit (KAFKA_MCP_MAX_SCAN + global limit) prevents unbounded decode loops — enforced in plan 02-03 consumer adapter, not here; this adapter decodes one message at a time |
| T-02-02-D | Elevation of Privilege | Protobuf dynamic class resolution | mitigate | If dynamic message type resolution is used, it must only resolve from Schema Registry schemas — no arbitrary Python import or eval; document the approach; raise DecodeError if class resolution fails rather than silently executing unknown code |
| T-02-02-SC | Tampering | new pip dependencies (fastavro/protobuf) | mitigate | confluent-kafka already declared in pyproject.toml and includes SR client + Avro support; if fastavro or google-protobuf are new additions, verify via pypi.org before installing; slopcheck gate applies |
</threat_model>

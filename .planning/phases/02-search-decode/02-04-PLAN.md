    GREEN — TopicService.search_messages:
    Signature: search_messages(self, key: str, *, key_field: str | None = None,
        topics: list[str] | None = None, time_from: datetime | None = None,
        time_to: datetime | None = None, limit: int = 500) -> list[KafkaMessage]

    Algorithm per CONTEXT.md:
    1. Resolve topics: if None → self._consumer.list_topics(include_internal=False)
    2. Resolve time_to: if None → datetime.now(tz=timezone.utc)
    3. Resolve time_from: if None → use OFFSET_BEGINNING (-2) directly (earliest)
    4. results: list[KafkaMessage] = []
    5. For each topic in topics:
       partition_ids = self._consumer.get_partition_ids(topic)
       For each partition_id:
         Resolve start_offset:
           If time_from is not None:
             ts_ms = int(time_from.timestamp() * 1000)
             start_offset = self._consumer.offsets_for_times(topic, partition_id, ts_ms)
           Else:
             low, _ = self._consumer.get_watermark_offsets(topic, partition_id)
             start_offset = low
         Resolve stop_offset:
           _, stop_offset = self._consumer.get_watermark_offsets(topic, partition_id)
         remaining = limit - len(results)
         if remaining <= 0: break (global limit reached)
         raw_msgs = self._consumer.fetch_messages(topic, partition_id, start_offset,
             stop_offset, time_to, remaining)
         For each raw_msg in raw_msgs:
           try:
             decoded = self._registry.decode(raw_msg.raw)
           except DecodeError:
             decoded = None  # resilient: value=None, keep message
           Build evidence_keys = extract_evidence_keys(decoded, raw_msg.headers)
           msg = raw_msg.model_copy(update={"value": decoded, "keys": evidence_keys})
           If key matches (per key_field):
             results.append(msg)
             if len(results) >= limit: break
       If len(results) >= limit: break
    6. Return results

    key_field matching function:
    _matches_key(msg: KafkaMessage, key: str, key_field: str | None) -> bool:
    - If key_field is None or key_field == "key": return msg.key == key
    - If key_field.startswith("header:"): name = key_field[7:]; return msg.headers.get(name) == key
    - If key_field.startswith("value:"): path = key_field[6:]; traverse dotted path in msg.value; return resolved == key
    - Else: return False

    Evidence extraction:
    _extract_evidence_keys(value: dict | None, headers: dict[str, str]) -> dict[str, str | None]:
    Well-known name map: order_id looks for "order_id"/"orderId"/"order-id" in value and headers;
    msisdn looks for "msisdn"/"phone"/"phoneNumber" in value and headers;
    customer_id looks for "customer_id"/"customerId" in value and headers;
    product_id looks for "product_id"/"productId" in value and headers.
    Returns dict with all four keys; absent identifiers are None.

    TopicService.__init__ extended: (self, consumer: ConsumerPort, registry: SchemaRegistryPort)
    Store self._registry = registry. All existing methods unchanged.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| domain service → ConsumerPort | Service calls fetch_messages/fetch_message; results are raw bytes — service must never exec or eval them |
| domain service → SchemaRegistryPort | Service calls decode(); result is dict | None; service trusts the adapter not to return executable objects |
| search_messages key matching | User-supplied key_field with "value:" prefix traverses decoded payload — dotted path traversal must be bounded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-A | Tampering | key_field "value:" path traversal | mitigate | _matches_key traversal uses simple dict key access on str-split path segments; raises KeyError (returns False for no match) rather than calling eval/getattr on arbitrary names; no code execution possible |
| T-02-04-B | Denial of Service | search_messages no-topic case | mitigate | topics=None calls list_topics() then iterates all; global limit bounds the result; if broker has 10k topics with 100 partitions each, max scan is limit × max_scan_per_partition — same cap as explicit topics |
| T-02-04-C | Information Disclosure | DecodeError propagation in get_message | mitigate | DecodeError carries topic/partition/offset/reason — reason is the adapter error message; adapter must not put sr_pass value in reason string (enforced in plan 02-02 adapter) |
| T-02-04-D | Denial of Service | limit=0 or negative limit edge case | mitigate | search_messages returns [] immediately when limit <= 0; get_message ignores limit (single message) |
| T-02-04-E | Information Disclosure | NullSchemaRegistry decode returns None | accept | NullSchemaRegistry is test/fallback only; decode returns None (no error), value=None on messages; no credentials involved |
| T-02-04-SC | Tampering | pip installs | accept | No new packages; all dependencies already in pyproject.toml from Phase 1 |
</threat_model>

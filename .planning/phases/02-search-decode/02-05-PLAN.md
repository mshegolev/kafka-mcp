<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → FastAPI | Untrusted JSON body; pydantic validates before reaching KafkaClient |
| MCP host → MCP stdio | Untrusted tool arguments; FastMCP type-validates before tool function executes |
| Serialized output → raw bytes | raw: bytes must be base64-encoded before leaving process boundary |
| CLI stdin → argparse | CLI arguments from shell; argparse validates types; datetime parsing uses fromisoformat() |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05-A | Tampering | SearchMessagesRequest.limit | mitigate | Pydantic field with max constraint: limit: int = Field(default=500, ge=1, le=10000); prevents limit=0 or limit=9999999 DoS; CLI uses argparse type=int with manual clamp or click validator |
| T-02-05-B | Information Disclosure | DecodeError.reason in HTTP 422 response | mitigate | reason is the adapter's error message — must not contain SR password; this is guaranteed by plan 02-02 adapter which does not put credentials in exception reason strings |
| T-02-05-C | Information Disclosure | raw bytes in serialized output | mitigate | All serialized faces (MCP, REST, CLI --json) base64-encode raw bytes before returning; in-process domain object retains bytes (never transmitted as bytes over HTTP/MCP) |
| T-02-05-D | Tampering | time_from / time_to ISO8601 injection | mitigate | Pydantic datetime fields in REST (auto-parsed); MCP uses datetime.fromisoformat() (stdlib, safe); CLI uses fromisoformat(); all fail with ValueError on malformed input (caught at face boundary) |
| T-02-05-E | Tampering | topics comma-split injection | accept | topics is a list of strings passed to list_topics/get_partition_ids; invalid topic names return TopicNotFoundError from broker (not code execution); Kafka topic names have no injection surface |
| T-02-05-F | Information Disclosure | SystemExit(2) stderr DecodeError reason | mitigate | CLI stderr shows reason from DecodeError — adapter must not put credentials in reason; guaranteed by plan 02-02 |
| T-02-05-SC | Tampering | pip installs | accept | No new packages; base64 and datetime are Python stdlib |
</threat_model>
